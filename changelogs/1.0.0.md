# 1.0.0 — StriMD

**FerrisMind fork** of [frostmark](https://github.com/Mrmayman/frostmark), renamed to **StriMD** (`strimd` on crates.io). Not compatible with upstream `0.3.x`.

Base upstream commit: `fbe35a6` (0.3.1 line: tables, ruby, iced 0.14).

## Added

### Public Cargo contract

- `no_iced` — headless builds without iced/GPU
- `static` — `Document::parse`, `Document::to_html()`, `HtmlFragment`
- `stream` — incremental LLM streaming via vendored **mdstream**

### Core API (backend-agnostic)

- `Document`, `RenderBlock`, `BlockId`, `BlockKind`, `BlockContent`, `BlockStatus`
- `ParseProfile`, `ParseOptions`, `RawHtmlPolicy`
- `StreamDocument`, `StreamOptions::chat()`, `StreamPatch`, `StreamUpdate`
- `ParseBackend`, `ParseDiagnostics`
- `markdown_to_html()` — pulldown static export (`static` feature)

### Markdown

- **pulldown-cmark 0.13** as the only Markdown parser (GFM tables, tasks, strike, footnotes, alerts, math, wikilinks)
- Compiled event cache for parse-once committed block rendering

### HTML

- `HtmlFragment`, `HtmlNode`, `HtmlTag`, `HtmlAttr`, `NodeId`
- html5ever **TreeSink → HtmlFragment** production path (no RcDom in iced backend)
- Raw HTML policies: preserve, escape, strip unsupported
- Optional `_html_preprocess` (`lol_html` rewrite layer)

### Iced backend

- `MarkState::from_document`, `from_blocks`, `sync_from_stream`
- `BlockRenderCache` + `DomRef` fragment traversal
- `MarkDocument` / `MarkSegment` / `CodeBlock` — prose/code split (pulldown-backed rich segments)

### Examples

- `static_export`, `stream_chat` (headless)
- `egui_table_harness`, `egui_pipeline_harness` (Tasks 4.5 / 7.4 validation)

### Tooling & docs

- `docs/API.md`, `docs/MIGRATION.md`, `docs/LEGACY_REMOVAL_GATE.md`, `docs/README_HEADLESS.md`
- `scripts/verify-features.sh`, `scripts/verify-all-features.sh`, `scripts/verify-egui-harness.sh`
- CI: default, headless, stream parity, HtmlFragment parity, egui harness, full feature matrix + clippy

---

## Changed

- Default build: iced + pulldown (no comrak)
- `MarkState::with_html_and_markdown` parses via `Document` / pulldown internally
- `markdown` Cargo feature alias now enables `static` (was `_legacy_comrak` on migration builds)
- List rendering improvements; text editor state removed from `MarkState`

---

## Removed

- **comrak** dependency and `_legacy_comrak` feature
- `LegacyFallbackPolicy`, `LegacyFallbackReport`, shadow-compare diagnostics
- `legacy_fallback_used()`, `shadow_mismatch()`, `BlockContent::LegacyHtml`
- Production **RcDom** path in iced backend (`_rcdom_compat` retained for parity tests only)

---

## Migration from upstream 0.3.x

```toml
# Headless (CI, export, LLM pipeline)
strimd = { version = "1.0", default-features = false, features = ["no_iced", "static", "stream"] }

# GUI (iced)
strimd = "1.0"
```

See [docs/MIGRATION.md](../docs/MIGRATION.md) for integration patterns.

**Known HTML delta:** `gfm_wikilink.md` — pulldown wikilink output differs from former comrak; pulldown is canonical.
